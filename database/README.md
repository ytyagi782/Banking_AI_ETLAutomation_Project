# database/ — new banking entities (6 x 4 layers)

This folder adds **6 new business entities**, each flowing through all four
layers (Source → PreStaging → Staging → DWH), plus the stored procedures that
load them. Everything here is generated by **`generate_schema.py`**, which is the
single source of truth — edit the entity definitions there and re-run it rather
than hand-editing the `.sql` files.

> The testing framework itself stays **READ-ONLY**. These scripts are DDL / load
> procedures you run **manually in SSMS** (like `GoldenTestData/10_full_reset_*`).
> The framework never executes them.

## The 6 entities and their DWH modelling

| Entity            | Source / PS / STG            | DWH table                | SCD / type |
|-------------------|------------------------------|--------------------------|------------|
| Employees         | SRC/PS/STG_Employees         | `DimEmployee_Type2`      | SCD **Type-2** |
| Loans             | SRC/PS/STG_Loans             | `DimLoan_Type2`          | SCD **Type-2** |
| Cards             | SRC/PS/STG_Cards             | `DimCard_Type1`          | SCD **Type-1** |
| Merchants         | SRC/PS/STG_Merchants         | `DimMerchant_Type1`      | SCD **Type-1** |
| CardTransactions  | SRC/PS/STG_CardTransactions  | `FactCardTransaction`    | **Fact** (→ DimCard, DimMerchant) |
| LoanPayments      | SRC/PS/STG_LoanPayments      | `FactLoanPayment`        | **Fact** (→ DimLoan) |

Naming matches the existing model: `SRC_`, `PS_`, `STG_` (with `IsValid` /
`RejectionReason`), `Dim<Entity>_TypeN`, `Fact<Entity>`, and
`usp_Load_<target>` load procedures.

## How to deploy (SSMS, one time)

Assumes the 4 `Bank_*` databases already exist.

**Option A — SQLCMD mode:** open `09_run_all_new_entities.sql` in SSMS, turn on
*Query → SQLCMD Mode*, and execute. It `:r`-includes every file in order.

**Option B — run individually** in this exact order:

1. `ddl/01_source_tables.sql`
2. `ddl/02_prestaging_tables.sql`
3. `ddl/03_staging_tables.sql`
4. `ddl/04_dwh_tables.sql`
5. `procs/05_load_prestaging.sql`
6. `procs/06_load_staging.sql`
7. `procs/07_load_dwh.sql`
8. `data/08_source_seed_data.sql`

Then load the data through the layers (Dims **before** the Facts that reference them):

```sql
USE [Bank_PreStaging];
EXEC dbo.usp_Load_PS_Employees;    EXEC dbo.usp_Load_PS_Loans;
EXEC dbo.usp_Load_PS_Cards;        EXEC dbo.usp_Load_PS_Merchants;
EXEC dbo.usp_Load_PS_CardTransactions; EXEC dbo.usp_Load_PS_LoanPayments;

USE [Bank_Staging];
EXEC dbo.usp_Load_STG_Employees;   EXEC dbo.usp_Load_STG_Loans;
EXEC dbo.usp_Load_STG_Cards;       EXEC dbo.usp_Load_STG_Merchants;
EXEC dbo.usp_Load_STG_CardTransactions; EXEC dbo.usp_Load_STG_LoanPayments;

USE [Bank_DWH];
EXEC dbo.usp_Load_DimEmployee_Type2;  EXEC dbo.usp_Load_DimLoan_Type2;
EXEC dbo.usp_Load_DimCard_Type1;      EXEC dbo.usp_Load_DimMerchant_Type1;
EXEC dbo.usp_Load_FactCardTransaction;  -- after DimCard + DimMerchant
EXEC dbo.usp_Load_FactLoanPayment;      -- after DimLoan
```

## Already wired into the test framework

You do **not** need to touch these again — the generator's output was merged in:

* `config/source_to_prestaging.yaml`, `prestaging_to_staging.yaml`,
  `staging_to_dwh.yaml` — 6 new table blocks each.
* `pytest.ini` — 6 new table markers (`employees`, `loans`, `cards`,
  `merchants`, `cardtransactions`, `loanpayments`).
* `tests/**` — 18 new `test_*` files (6 entities × 3 layers).

Once the tables exist and the load procs have run, validate them with:

```bash
pytest -m "employees or loans or cards or merchants or cardtransactions or loanpayments"
```

## Regenerating

Edit the `DIMENSIONS` / `FACTS` lists in `generate_schema.py`, then:

```bash
.venv/Scripts/python.exe database/generate_schema.py
```

This rewrites the `.sql` files, the `tests/**` files, and the
`_config_additions/` snapshots. If you change columns/keys/rules, re-merge the
snapshots into `config/*.yaml` and `pytest.ini` by hand (the generator does not
overwrite those, to protect the existing 4 entities' hand-tuned rules).
